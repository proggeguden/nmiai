# NM i AI — Project Rules

## Workflow
- After implementing and testing a feature, always create a git commit.
- Test locally before committing. Don't commit broken code.

## Project Structure
Each competition task lives in its own folder (e.g., `tripletex/`, `norgesgruppen/`, `astar-island/`).
Each task folder has its own `CLAUDE.md` with task-specific context.

## Competition Tasks
- **Tripletex** (`tripletex/`) — AI accounting agent, POST /solve endpoint
- **NorgesGruppen** — object detection (mAP@0.5)
- **Astar Island** — Norse world prediction
