## POST /ledger/voucher
Add new voucher. IMPORTANT: Also creates postings. Only the gross amounts will be used. Amounts should be rounded to 2 decimals.

### Prerequisites
- Look up account IDs first: GET /ledger/account?number=NNNN (account numbers are NOT IDs)

### Query Parameters
| Param | Type | Required | Description |
|-------|------|----------|-------------|
| `sendToLedger` | boolean | no | Should the voucher be sent to ledger? Requires the "Advanced Voucher" permission. |

### Send Exactly
```json
{
  "date": "YYYY-MM-DD",
  "description": "<string>",
  "postings": [
    {
      "account": {"id": <int>},
      "amountGross": <positive_for_debit>
    },
    {
      "account": {"id": <int>},
      "amountGross": <negative_for_credit>
    }
  ]
}

```

### Optional Fields
| Field | Type | Description |
|-------|------|-------------|
| `date` | string |  |
| `description` | string |  |
| `externalVoucherNumber` | string | External voucher number. Maximum 70 characters. |
| `id` | integer(int64) |  |
| `postings` | array(Posting) |  |
| `reverseVoucher` | ref(Voucher) |  |
| `vendorInvoiceNumber` | string | Vendor invoice number. |
| `version` | integer(int32) |  |
| `voucherType` | ref(VoucherType) | Voucher type. Must not be of type 'Utgående faktura' ('Outgoing Invoice') on new... |

### DO NOT SEND
- `attachment` (read-only)
- `changes` (read-only)
- `displayName` (read-only)
- `document` (read-only)
- `ediDocument` (read-only)
- `number` (read-only)
- `numberAsString` (read-only)
- `supplierVoucherType` (read-only)
- `tempNumber` (read-only)
- `url` (read-only)
- `wasAutoMatched` (read-only)
- `year` (read-only)
- `number` — auto-generated by system
- `voucherType` — do NOT send — system determines type

### Common Errors
| Symptom | Fix |
|---------|-----|
| 422 null postings | postings array is REQUIRED and must contain at least 2 entries |
| 422 postings don't balance | Debit = positive amount, credit = negative amount. Sum must be 0. |
| 422 VAT amount mismatch | When posting has vatType, use GROSS amount — Tripletex auto-generates the VAT line |

> ⚠️ Account numbers (e.g. 1920, 2400) are NOT the same as account IDs — always GET /ledger/account?number=X first

> ⚠️ Debit = positive amountGross, credit = negative amountGross

> ⚠️ If posting includes vatType, Tripletex auto-generates the corresponding VAT posting

> ⚠️ Known vatType IDs (no lookup needed): 1=0%, 3=25%, 5=15%(food), 6=12%(transport)

## POST /ledger/voucher/{voucherId}/attachment
Upload attachment to voucher. If the voucher already has an attachment the content will be appended to the existing attachment as new PDF page(s). Valid document formats are PDF, PNG, JPEG and TIFF. Non PDF formats will be converted to PDF. Send as multipart form.

### Path Parameters
- `voucherId`: integer **(required)**

### Send Exactly
```json
{
  "file": "<string>"
}
```
